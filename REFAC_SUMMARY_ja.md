# KabuRadar 整理・リファクタリング内容

## このワークスペースに保存したもの

- `KabuRadar_refactored/` を新規作成
- 元プロジェクトを直接変更せず、改善版の骨組みを分離保存

## 主な整理ポイント

1. **機密情報分離**
   - `.env.example` を追加
   - LINEトークン/ユーザーIDをソース直書きしない実装へ変更

2. **パス依存性の改善**
   - `C:\...` 固定パス依存を排除
   - プロジェクトルートから相対解決する `launcher.py` を追加

3. **起動フローの吸収**
   - `2-1...` / `2-1..._GetYahooF` など複数命名を候補実行に統一
   - 時間帯分岐を1か所に集約

4. **依存管理の明文化**
   - `requirements.txt` を追加
   - 最低限必要な外部ライブラリを列挙

5. **設定管理の整理**
   - `config/config_lo.ini` を配置（短期RSI / LO モード）
   - `src/kaburadar/config.py` で `ini + env` を共通読み込み

## 次の推奨作業

- 元 `software/src` から必要スクリプトを `src/kaburadar` 配下へ段階移植
- DB初期化スクリプトを作成して `DB/` 空問題を解消
- `pytest` による最小テスト（設定読み込み、ランチャー分岐）を追加
